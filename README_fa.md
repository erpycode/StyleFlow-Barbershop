# StyleFlow Barbershop 💈 (By ErPyCode)

یک اپلیکیشن مدرن و فول‌استک برای مدیریت نوبت‌دهی آرایشگاه، طراحی شده با تمرکز بر تجربه کاربری (UX) و رابط کاربری (UI). این پروژه به صورت دوزبانه (فارسی/انگلیسی) و با قابلیت ذخیره‌سازی محلی برای دمو توسعه یافته است.

## 🚀 ویژگی‌های کلیدی
- **پشتیبانی از دو زبان (i18n):** قابلیت سوئیچ بین زبان‌های فارسی (RTL) و انگلیسی (LTR) در تمام بخش‌ها.
- **امضای اختصاصی:** استفاده از برند شخصی **ErPyCode** در جای‌جای اپلیکیشن.
- **ذخیره‌سازی هوشمند محلی:** استفاده از LocalStorage برای حفظ داده‌ها بدون نیاز به دیتابیس (ایده‌آل برای پورتفولیو).
- **دستیار هوشمند (AI):** بهره‌گیری از هوش مصنوعی Gemini برای پاسخگویی به سوالات مشتریان.
- **پنل مدیریت پیشرفته:** مدیریت کامل خدمات، مشاهده نوبت‌ها و تنظیمات آرایشگاه.

## ✒️ امضای دیجیتال و کپی‌رایت

## 🛠 تکنولوژی‌های استفاده شده
- **Frontend:** React.js, Vite, Tailwind CSS
- **State & Storage:** LocalStorage API
- **AI:** Google Gemini API
- **Animations:** Framer Motion
- **Icons:** Lucide React

## 📦 نصب و راه‌اندازی سریع

۱. پروژه را کلون کنید:
```bash
git clone https://github.com/ErPyCode/styleflow-barbershop.git
```

۲. وابستگی‌ها را نصب کنید:
```bash
npm install
```

۳. فایل `.env` را ساخته و کلید Gemini را در آن قرار دهید:
```env
GEMINI_API_KEY=your_key_here
```

۴. پروژه را اجرا کنید:
```bash
npm run dev
```

## 🗄 راهنمای اتصال به دیتابیس (Real Database)
به صورت پیش‌فرض، این پروژه از `LocalStorage` برای ذخیره اطلاعات استفاده می‌کند تا نیازی به راه‌اندازی سرور نداشته باشید. اگر می‌خواهید آن را به یک دیتابیس واقعی (مثل Supabase) وصل کنید:

1. فایل `supabaseClient.ts` را باز کرده و کد مربوط به راه‌اندازی کلاینت سوپابیس را قرار دهید.
2. در `services/dataService.ts` توابع دیتابیس را جایگزین توابع LocalStorage کنید (کد قبلی در تاریخچه در دسترس است).
3. جداول `bookings` و `services` را با ساختاری که در بخش جداول آمده بسازید.

## 🎨 شخصی‌سازی و تغییرات
- **حذف صفحه درباره پروژه:** برای حذف این صفحه، کافیست مسیر `/about` را از `App.tsx` حذف کرده و دکمه‌های مربوطه را در فایل `components/Layout.tsx` کامنت کنید.
- **تغییر اطلاعات تماس:** تمامی لینک‌های شبکه اجتماعی در فایل `pages/AboutProject.tsx` قرار دارند.
- **تغییر لوگو و متن‌ها:** برای تغییر نام پروژه، به فایل `services/LanguageContext.tsx` مراجعه کنید.
